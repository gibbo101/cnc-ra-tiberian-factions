## ⚠️ LESSONS (read first — this session went badly by ignoring them)

1. **NEVER owner-open a shared RA production building for a faction.** A building yields its OWN
   faction's roster (helipad TDHPAD works *because* it's a separated GDI building). Owner-opening
   AFLD/SYRD/SPEN made GDI/Nod inherit the Soviet/Allied unit lists. → Separated `STRUCT_TD*` buildings.
2. **Every TD entity = its own engine type + its own COPIED HD art files.** Never
   share frames (Image=SYRD, or a tileset pointing at another unit's `ss\` frames) — a later
   faction-logo reskin would corrupt the RA original. Copy the art into `TD*`-named files
   (scripts/bundle_ra_building.py for HD). **The classic SHP port is NO LONGER required** — classic
   mode is scrapped (see §2c + [[project-classic-mode-scrapped]]); new entities ship HD only and
   fall back to the RA donor sprite in classic, which is accepted. (Older entities that already have
   a `build_tfassets.sh` classic entry keep it; just don't add new ones.)
3. **Separating a building is NOT just cloning the ctor.** The recipe's per-building work is
   MANDATORY and was skipped this session, causing every building bug:
   - **`_anims[]` table** (bdata.cpp One_Time) — per-`STRUCT` animation states. No entry = static.
   - **Per-type dispatch (gotcha #5)** — the engine hardcodes `STRUCT_SHIP_YARD`/`SUB_PEN`/`AIRSTRIP`
     in ~30 sites (vessel exit, dock, repair, fixed-wing landing, the `STRUCTF_AIRSTRIP` BScan flag).
     A new `STRUCT_TD*` must be added to every applicable one. `TDAFLD` is the airstrip precedent.
   - **Prereq tokens:** units must use the **RA token** (`syrd`/`spen`/`afld`) + a `house.cpp`
     Can_Build equivalence remap — because `BuildingTypeClass::From_Name` only resolves
     `< STRUCT_COUNT`, so a direct `tdgyard` prereq token silently fails.
4. **Don't claim "done" without the dispatch + anims.** "Compiles + appears in sidebar" ≠ works.
